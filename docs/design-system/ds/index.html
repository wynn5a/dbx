Data Buddy Design System v1.1

The database client
design system.

Extracted from the Data Buddy product. A dense, calm, keyboard-first dark interface in the Linear lineage — built for people who live in their data. Hierarchy from elevation, color reserved for meaning, type that tells UI from data at a glance.

Principles
01Dark & layered

Five near-black surfaces stack to build depth. Borders are 5–12% white hairlines, never solid lines. The page stays flat; only overlays cast a shadow.

02Color means something

Neutral by default. The accent marks primary actions and focus; semantic hues signal state; every data type owns a fixed color reused everywhere.

03Dense & quiet

34px rows, tight padding, small type. Information-rich without noise — chrome recedes so the data is the loudest thing on screen.

04UI vs. data type

Geist carries the interface; Geist Mono carries anything machine-shaped. Tabular numerals keep every column honest.

05Keyboard-first

⌘K reaches everything. Visible key hints, focus rings, and a command palette make the mouse optional.

06Tweakable surface

Accent, font, density, radius and sidebar width are tokens the user can bend — the system holds together across the range.
